    Digital Twin

    So what is a digital twin? It is quite popular now especially when a lot of data is needed to create a model. It is basically a model that can behave as the actual system when real inputs are introduced to the model. A digital twin can be a physical model or a machine learning model. But it doesn't make sense to use a slow-running physical model to do a quick run to test some scenarios or failures in real-time. So, a machine learning type model or surrogate model is used to represent the physical model which can run really fast. The model can be coupled with subsystems for more accurate predictions.

    Digital Twin

    The Transformer

    Self-attention Mechanism
    The animation below illustrates how they apply the Transformer to machine translation from english to french. Neural networks for machine translation typically contain an encoder reading the input sentence and generating a representation of it. A decoder then generates the output sentence word by word while consulting the representation generated by the encoder. The Transformer starts by generating initial representations, or embeddings, for each word. These are represented by the unfilled circles. Then, using self-attention, it aggregates information from all of the other words, generating a new representation per word informed by the entire context, represented by the filled balls. This step is then repeated multiple times in parallel for all words, successively generating new representations. The decoder operates similarly, but generates one word at a time, from left to right. It attends not only to the other previously generated words, but also to the final representations generated by the encoder.

    Different Types of Learning

    Self-supervised Learning

    • Aims at creating a data-efficient artificial intelligent system
    • Self-supervised learning does not focus on clustering and grouping
    • An autonomous form of supervised learning
    • Use only unlabelled data to find underlying data structure
    It is quite an unusual way of learning by just relying on unlabelled data. It is quite a new concept and is getting more attention.

    Federated Learning

    Federated Learning

    Federated Learning

    • A global model distributed to the clients to enhance the model
    • Only new weights are returned to the federated server
    • Returned weights are incorporated into global model
    Federated learning is a learning technique that train ML models across multiple devices. The local models are trained locally and the parameters are sent back to the federated server to be combined with a global model. It can provide more customised services for customers. I know some of you will argue about data privacy.

    Federated Learning

    Is privacy an issue?

    • Private user data are only processed locally
    • Only new weights are shared
    • Model weights are values that is hard to deduce and is just model parameters
    • It will not only help to eliminate data security and privacy issues but customised services for every end-user
    • Used by King's College London and American College of Radiology for medical imaging and radiology
    But it is really an issue? Since only parameters are being transferred across the network to the federated server, it is actually safer because data are being processed locally. Data are kept on the edge devices and removed after use. Model parameters looks like random number and by itself it means nothing. Because it is quite safe, King's college london is using it for medical imaging.

    Learning Using Privileged Information: SVM+

    Next, is about LUPI or learning using privileage information.

    What is SVM?

    Support Vector Machine (SVM)

    • SVM or Support Vector Machine
    • A classifier and it can be a linear or non-linear classifier
    • Performs classification by finding the hyperplane that maximises the margin between classes
    Before getting into LUPI, we need to look at SVMs. SVM or support vector machine is a type of classifier. It performs classification using a hyperplane. The hyperplanes are the boundaries found by maximising the margin between classes.

    Learning using Privileged Information

    What is LUPI?

    • Teacher-student based learning
    • Humans are effective at learning with the help of a teacher
    • Teacher share comments, explanations and examples to facilitate learning process
    • LUPI uses additional information in training phase (data only available in training phase)
    LUPI is a teacher-student based learning. It takes advantage of using lots of information that is only available during training process. Similar like students learning at school. Teacher shares experience and explanations to facilitate learning process.

    Learning using Privileged Information

    SVM+

    • SVM+ is an SVM that uses additional information to learn
    • It estimates the slack variables and use transfer learning for new models
    • Benefits:
      • Faster convergence
      • Fewer training examples needed
    What LUPI does is train the slack variables in SVM. These makes the SVM into a new type of SVM called SVM+. Slack variables are variables that allow soft margins. It is using transfer knowledge to bring the learnt experiences to the a classifier. But this only happens during training process where a lot of data is available. It helps for faster convergence compared to the conventional method.

    TinyML

    TinyML

    • A type of machine learning that fits into tiny hardware
    • Where machine learning meets edge computing
    • Reduced and optimised full stack ML that can run in small devices
    • Makes it possible to get inference without sending data to cloud
    • Applications includes voice commands, image processing, detection, monitoring and many more
    • Next AI revolution
    TinyML is a field of study in Machine Learning and Embedded Systems that explores the types of models you can run on small, low-powered devices like microcontrollers. Sometimes, big is not always better. Simple problems can be solve using small models. On advantage of tinyML is privacy. It runs prediction in the device, so data is not stored in any server.

    TinyML Applications

    TinyML Applications

    • A reinforcement learning agent is combined with a PID controller for airpath control
    • The DQN controller is also able to compensate if the PID controller has errors due to software faults
    These are some applications of tinyML. Because of the small form factor, it allows us to apply ML to almost anywhere.
